# Personal Git Report

🌟 打造属于你的年度编程回顾报告

## 项目简介

Personal Git Report 是一个轻量级的个人Git年度报告生成器，专门为开发者创建精美的编程年度总结。它能够：

- 📊 **生成年度总结报告** - 类似GitHub年度回顾的专业报告
- 🎯 **统计编程成就** - 提交次数、代码行数、项目数量等关键指标
- 📈 **可视化展示** - 精美的图表和动画效果
- 🎨 **个性化定制** - 支持多种主题和样式

### 为什么需要这个工具？

厌倦了GitHub的简单统计？想要一个更详细的个人编程年度回顾？这个工具帮你创建：

- 类似 **GitHub Year in Review** 的精美报告
- 包含详细的技术栈分析和成长轨迹
- 可导出分享的PDF/HTML格式
- 支持多个项目和仓库的聚合分析

## 🚀 快速开始

### 一键安装
```bash
# 下载并运行（推荐）
npx create-git-report

# 或者全局安装
npm install -g personal-git-report
```

### 基础使用（3种方式）

#### 方式1：快速分析当前目录
```bash
git-report
```

#### 方式2：指定年份生成报告
```bash
git-report --year 2023
```

#### 方式3：分析多个项目
```bash
git-report --projects "/my-project-1,/my-project-2"
```

### 高级选项（可选）

```bash
# 完整参数示例
git-report \
  --year 2023 \                    # 指定年份
  --theme dark \                   # 选择主题 (light/dark/colorful)
  --format pdf \                   # 输出格式 (html/pdf/json)
  --output ./my-report \           # 输出目录
  --author "your-email@example.com" # 指定作者
```

## ⚙️ 配置选项（可选）

### 简单配置
创建 `git-report.config.js` 文件：
```javascript
module.exports = {
  // 基本信息
  author: "你的昵称",
  email: "your-email@example.com",

  // 主题设置
  theme: "dark",           // light/dark/colorful

  // 显示选项
  showPrivateRepos: false, // 是否显示私有仓库
  excludeWeekends: true,   // 是否排除周末

  // 输出设置
  output: "./my-report",
  format: ["html", "pdf"]
};
```

### 高级自定义
```javascript
module.exports = {
  // 自定义时间范围
  dateRange: {
    from: "2023-01-01",
    to: "2023-12-31"
  },

  // 排除规则
  exclude: {
    paths: ["node_modules", "*.min.js"],
    commits: ["Merge pull request", "Update dependencies"]
  },

  // 自定义统计
  customStats: {
    countLines: true,
    analyzeComplexity: true,
    trackLearning: true
  }
};
```

## ❓ 常见问题

### 🚀 快速问题
**Q: 第一次运行很慢？**
A: 首次分析需要扫描所有Git历史，后续会使用缓存加速。

**Q: 支持哪些Git平台？**
A: 支持任何Git仓库：GitHub、GitLab、Gitee、自建Git服务器等。

**Q: 能分析他人仓库吗？**
A: 可以分析任何你有访问权限的仓库，但建议只分析自己的数据。

**Q: 报告数据准确吗？**
A: 基于Git日志数据，非常准确。排除merge提交后更接近实际编码量。

### 🔧 技术问题
**Q: 如何处理大型仓库？**
A: 工具会自动优化大仓库的分析速度，也可以使用 `--shallow` 参数只分析部分历史。

**Q: 内存占用太高？**
A: 可以使用 `--low-memory` 模式，会牺牲一些速度换取低内存占用。
